## 2. MCP之詢問技巧

### 2.1 LLM之選擇

- 原則1：外部模型會比地端更聰明，適合複雜、繞彎的計算。地端模型適合簡單的查詢。
- 原則2：Anthropic模型會顯示計算每個過程的邏輯與結果，可以逐步檢查。
### 2.2 範例

- 先以簡單的問題開始，發現AI回應有錯，需要導正，則加上新的原則。
- 逐步累積原則，直到正確、滿意為止。

```python
"""
請經由google-calendar-mcp統計環評組、大中興碳管理組、廠站組、空氣組、管線組(中興+環興)
在2025/11/19(星期三)開始未來2周(不含周末)，哪一天請假、外出開會的人最少?
請假包括註記為"休"、"假"等等，
外出包括："現勘"、"開會"、"月會"等等所有可能的外部會議。
有的時候同仁會把同一個會議出席的人，用括弧簡稱來註記，如"1728C-拜訪李○○老師(家琪、姿佑、中興)"這樣有3人出去開會不在，
同理"辰、逸、謝、樵-楠○細設審查會"有4人不在公司。
不要忽略了google calendar全天的行程。
請按這些原則進行統計。
"""
```

- MCP執行畫面：會有協定的{鍵：值}配對組，可供檢查、確認。

### 2.3 步驟

1. 我的google-calendar-mcp權限可以看到哪些日曆?
2. 今天是2025/11/19(星期三)開始未來2周(不含周末)，有哪些人請假?
3. 所謂的請假，包括註記為"休"、"假"等等事件，請重新查詢。
4. 這2周內，每天有幾個人外出開會?
5. 外出包括："現勘"、"開會"、"月會"等等所有可能的外部會議，請重新查詢。
6. 有的時候同仁會把同一個會議出席的人，用括弧簡稱來註記，如"1728C-拜訪李○○老師(家琪、姿佑、中興)"這樣有3人出去開會不在，同理"辰、逸、謝、樵-楠○細設審查會"有4人不在公司。重新計算人數。
7. 不要忽略了google calendar全天的行程。重新計算人數。

## 5. 名詞解釋

### MCP

Model Context Protocol (MCP) 是一個由 Anthropic 開發的開放標準，旨在標準化應用程式如何向大型語言模型 (LLM) 提供上下文的情境資訊。它就像 AI 應用程式的 USB，允許 LLM 安全且標準化地與外部資料、應用程式和服務互動。MCP 於 2024 年推出，目的是簡化 LLM 與外部世界的整合，使其能夠更容易地存取和處理資訊。

![](https://miro.medium.com/v2/resize:fit:1100/format:webp/1*VEJFbzlFbVPimfEI92tfbw.png)

### Smithery.ai

Smithery.ai 是一個為開發人員提供的平台，主要功能是協助尋找和部署與模型上下文語境協議規格相容的語言模型擴展。它也為代理程式配備了雲端電腦和大量工具，且不會超出上下文視窗限制。此平台可以協助使用者輕鬆地將 MCP 伺服器安裝到他們的 Claude Desktop 或其他類似應用程式中。

![](https://miro.medium.com/v2/resize:fit:1100/format:webp/1*tm-7iH-8Ib6Og4WnC5orww.png)
